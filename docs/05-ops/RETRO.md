# RETRO — v1.4.0 후속 정비 (2026-06-23)

> v1.4.0 배포 이후 같은 날 진행한 유지보수 회고. 새 버전이 아니라 문서 정합·환경·미러 품질 정비.

## 무엇을 했나

정본 대시보드 stale 정정 → 문서 전수 일관성 점검·동기화(living/snapshot 구분) → 자동 커밋 훅 수정 → 미러(깃털 뱀) 강화·repo 추적 → 대시보드(정본+미러+라이브) 동기화. 커밋 `6585263`·`5f45772`·`9a9f581`·`50e67fe`·`456dab7`, 전부 push·CI green.

## 잘된 점

- **사용자 지적이 전수 점검을 촉발**: "USAGE가 예전 버전"이라는 한마디로 living 문서 6종의 stale을 적발. 프로젝트 자신의 교훈("현재성 vs 스냅샷")을 실제 적용 — living만 동기화, snapshot은 동결.
- **"git status clean 미스터리"를 추측으로 넘기지 않음(§15)**: working tree가 비어 보이는 모순을 파고들어 **전역 Stop 자동 커밋 훅**을 발견. generic 메시지가 CONVENTIONS와 충돌함을 정직하게 표면화하고, 사용자 결정 후 WIP 체크포인트로 수정.
- **안전 게이트 존중**: `c2362a9` 메시지 교체(=히스토리 재작성)를 §1.6 게이트로 보고 임의 실행하지 않음. 승인 후, interactive rebase 미지원 환경에서 soft-reset+재커밋으로 깨끗한 Conventional 히스토리 복구(전부 미푸시라 안전).
- **렌더를 실제로 검증(§14)**: Playwright MCP가 headed로 떠 실패하자 Chrome headless로 우회해 desktop·mobile 캡처 — "확인했다"를 스크린샷 증거로 뒷받침.
- **미러 stale 회피 실천**: 정본을 고칠 때마다 미러도 동기화·같은 URL redeploy. 미러 소스를 `docs/assets/`로 추적해 재현성 확보.

## 아쉬운/배운 점

- **같은 stale이 또 반복됨**: v1.4.0 사이클 종료 시 living 문서(USAGE·RUNBOOK·SECURITY·docs/README)가 stale로 남아 있었다. v1.3.0 RETRO가 이미 "사이클 종료 시 현재성 문서 동기화"를 교훈으로 적었는데도 재발. → **개선: Ship 체크리스트(§12)에 "living 문서 동기화"를 명시 항목으로(차기).**
- **환경 훅을 늦게 파악**: 전역 자동 커밋 훅을 미리 몰라 "변경이 사라진 듯한" 혼동·조사 비용 발생. → 메모리(`auto-commit-stop-hook`)에 기록해 다음 세션이 즉시 인지.
- **미러 소스가 비추적이었음**: scratchpad/아티팩트에만 존재해 재현 불가였음 → repo 추적으로 해소.

## 잘된 점

- **실제 Claude–GPT 교차검증을 처음 실현**: 로컬 Codex(gpt-5.5, xhigh)로 §5를 시뮬레이션이 아닌 실제로 수행. v1.2.0 이래 "GPT 미접속"(§5.6·D7) 한계 해소.
- **교차검증이 값을 했다(실증)**: 단일모델 적대검토(F1~F6)가 놓친 **§1.6 구멍**("보조 검토가 사람 게이트를 대체하는 것처럼 오독")을 GPT가 적발 → 배포 전 반영. "증명→뒷받침"·재개 재검증·증거 위생도 GPT 신규.

- **게이트 존중**: 조사·docs·본문 패치는 자율, 배포(tag/Release)는 사람 승인 후. "GPT 먼저"를 배포 직전 실제 수행.

## 아쉬운/배운 점

- **본문 ↔ P2 축소 긴장**: 새 패턴이 Core Contract 축소 목표와 충돌 → 최소(§18.7 ~7줄)만 넣고 나머지는 P2로. GPT도 누락 패턴(capability discovery·2계층 메모리 규칙)을 P2 로드맵으로 독립 확인.
- **미러 live redeploy 미완**: 기존 artifact URL 부재로 같은 URL redeploy 불가 — 소스만 동기. 차기: URL을 repo/메모리에 기록.
- **Codex 비-git 디렉터리**: `codex exec`에 `--skip-git-repo-check` 필요.
