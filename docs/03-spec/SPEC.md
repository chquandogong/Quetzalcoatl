## 비기능 요구사항

- 보안: §1.6 게이트 불변, 자율로 약화 금지.
- 정직성: 강제 안 되는 메커니즘을 "보장"으로 표기 금지(§15).
- 호환성: 기존 §0~§17·모드 표 보존. frontmatter ≤ 1024자(현재 856).
- 모델 중립: 파일/git/artifact 없는 환경서 텍스트로 강등.

## 엣지 케이스 / 실패 처리

- 기존 repo에 `git init` 시도 → 가드로 확인.
- 레이트리밋 중 부분 커밋 → 재개 전 중복 확인.
- 동시 claim → git 충돌로 표면화, 먼저 안착 커밋 우선.
- ChatGPT서 Mermaid 미렌더 → 코드블록 텍스트로 수용.

## 사람 승인 게이트

배포(푸시·태그)는 사전승인됨(A2). 플러그인 재설치 시점은 사람 판단.

## 미해결 질문

실사용 멀티에이전트 동시 작업 빈도, ChatGPT 경로의 수동 재개 부담.
