# GLOSSARY — 용어집

> 상태: 활성 · 날짜: 2026-06-23 · 소유자: Documentarian · 승인: CHENGHAO QUAN

SKILL.md §22 부록 A와 동기화. 용어는 여기로 모은다(§6.2).

| 용어                             | 뜻                                                                                                                                                                            |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 쐐기(Wedge)                      | 가장 좁은 사용자의 가장 아픈 문제를 푸는 가장 작은 해결책. 1주 내 검증 단위.                                                                                                  |
| 안전 게이트(Gate)                | 사람 승인 없이 자동 실행하지 않는 비가역·외부·고위험 작업(SKILL §1.6).                                                                                                        |
| 자율 수준(Autonomy Level)        | L0 신중 / L1 표준 / L2 적극 / L3 완전. 확인 질문을 어디까지 생략할지의 **약속**.                                                                                              |
| 실행 허용목록(Allowlist)         | 자율 세션에서 확인 없이 실행 가능한 경로/명령/행동의 명시 목록(§18.2).                                                                                                        |
| 승인된 계획 범위 내 자율         | 사람이 방향을 1회 승인한 뒤 그 범위 안에서만 자율 실행하는 모델(§1.7, D1).                                                                                                    |
| 체크포인트(Checkpoint)           | 안전하게 재개할 수 있는 마지막 지점 = 마지막 성공 커밋(§19.1).                                                                                                                |
| 재개(Resume)                     | 중단·레이트리밋 후 체크포인트부터 무손실로 이어가기(§19).                                                                                                                     |
| 핸드오프(Handoff)                | 다른 에이전트/기기가 작업을 이어받도록 공유 상태를 넘기는 것(§21).                                                                                                            |
| 작업 보드(claimable board)       | todo/doing/done/blocked + 소유자를 가진 작업 목록. 나눠·이어서 작업의 단위(§21.1).                                                                                            |
| commit-as-CAS                    | 커밋 안착으로 claim을 확정하는 방식. 동시 claim을 git 충돌로 표면화(§21.2).                                                                                                   |
| 단일 라이터(Single-writer)       | 공유 보드에 한 주체만 쓰게 해 동시쓰기 손상을 막는 패턴(§21.2).                                                                                                               |
| 단일 진실 원천(SSOT)             | 상태가 갈라질 때 기준이 되는 곳. 본 스킬에서는 git repo(§21.3).                                                                                                               |
| live artifact                    | claude.ai에 렌더되는 공유 가능 페이지. 핸드오프에서는 **읽기 미러**(§21.3).                                                                                                   |
| 결정 대기                        | 사람 결정이 필요해 실행을 보류한 항목 묶음(§7, §18.3).                                                                                                                        |
| 모델 중립 강등                   | 파일/git/artifact가 없는 환경에서 같은 규율을 마크다운 텍스트로 대체(§18.6 등).                                                                                               |
| 게이트 priors                    | 모델이 학습으로 이미 갖춘, 파괴적 작업을 꺼리는 경향. 자율 절이 이를 대체하지 않음.                                                                                           |
| 앵커된 선호(Anchored Preference) | 사용자가 제시한 방향 선호/기본값(prior). 근거가 이길 때만 투명하게 뒤집고, 근소하면 선호 유지(tie→user). 맹종↔독단의 중도(§4.2).                                              |
| 강제 바인딩(Enforcement Binding) | 안전 규율(게이트·체크포인트·하위 에이전트 범위)을 환경이 제공하는 강제 수단에 연결하는 것. **있을 때만**이며 스킬이 작동을 보장하지 못함 — 확인 전 "강제됨" 가정 금지(§18.7). |
| 프로젝트 브리핑(`/context`)      | 프로젝트 현재 상태를 벤더-중립 1회성 읽기 스냅샷으로 합성해 새 모델·에이전트 환경을 온보딩. SSOT(repo·docs)에서 export하며 핸드오프(§21)와 구분(§24).                         |
