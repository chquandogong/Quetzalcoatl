# CONVENTIONS — 문서·Git 규칙 (단일 기준)

> 상태: 확정 · 날짜: 2026-06-23 · 소유자: Documentarian · 승인: CHENGHAO QUAN

"우선 룰을 정하고 일관되게 적용한다"(요청 ③). 이 파일이 **단일 기준**이며, SKILL.md §6/§20/§22와 동기화한다. 혼용 금지(§15).

## 1. 문서 규칙

| 항목      | 규칙                                                                                                 |
| --------- | ---------------------------------------------------------------------------------------------------- |
| 위치      | `docs/00-overview · 01-discovery · 02-decisions · 03-spec · 04-quality · 05-ops · appendix · assets` |
| 형식      | 전부 마크다운(.md)                                                                                   |
| 표현      | **그림(Mermaid)·표 우선**, 서술은 보조                                                               |
| 용어      | 정의는 `appendix/GLOSSARY.md`로 집중                                                                 |
| 머리말    | 모든 문서 첫 줄: `> 상태: … · 날짜: YYYY-MM-DD · 소유자: … · 승인: …`                                |
| 파일명    | 표준 문서는 대문자 스네이크(`DECISION_LOG.md`), 디렉터리는 숫자 접두(`02-…`)                         |
| 상태값    | 초안 / 활성 / 확정 / 보류 / 폐기                                                                     |
| 단일 체계 | 루트 평면 배치와 `docs/` 트리를 **섞지 않는다**                                                      |

## 2. Git 규칙

| 항목      | 규칙                                                                                      |
| --------- | ----------------------------------------------------------------------------------------- |
| init      | 기존/상위 repo면 확인 후. 이름=정규화 프로젝트명(소문자 kebab, ASCII)                     |
| 커밋 형식 | Conventional Commits: `<type>(<scope>): <요약>` (feat/fix/docs/refactor/test/chore/build) |
| 커밋 단위 | **의미 있는 단위**마다(마이크로스텝 커밋 금지)                                            |
| 트레일러  | AI 작업은 `Co-Authored-By:` 트레일러                                                      |
| 참조      | 결정 커밋은 본문에 `DECISION_LOG`/`CROSS_VALIDATION_LOG` 참조                             |
| 푸시      | 원격 존재 + 사용자 허용 시. **새 공개 원격 생성은 게이트**(§1.6)                          |
| 태그      | Ship 시 주석 태그 `vX.Y.Z`(SemVer). **명명 일관**(혼용 금지)                              |

## 3. 본 프로젝트 적용 예 (이번 사이클)

- 커밋 타입: `feat`(능력 추가), `docs`(문서·README·CHANGELOG).
- 태그: `v1.2.0`(MINOR — 하위호환 기능 추가).
- 과거 태그 비일관(`v1.1.0` vs `quetzalcoatl--v1.1.0`)은 **보존**하고, 앞으로 `vX.Y.Z`로 표준화.

## 4. 환경별 강등

git·파일·live artifact가 없으면(예: ChatGPT 일반 챗) 위 규칙을 **텍스트 산출물**로 대체한다: 변경 요약, 버전/체크포인트 메모, 붙여넣는 대시보드 텍스트.
