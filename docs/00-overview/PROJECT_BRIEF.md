# PROJECT BRIEF — Quetzalcoatl v1.2.0

> 상태: 확정 · 날짜: 2026-06-23 · 소유자: Quetzalcoatl OS · 승인: CHENGHAO QUAN

## 문제

Quetzalcoatl v1.1.0은 **한 세션 안에서 의미를 검증하고 자문**하는 데 강하지만, 실제 실행을 길게 끌고 갈 때 약했다.

- 매 단계 확인을 받느라 긴 작업이 끊긴다.
- 레이트리밋·세션 종료에 작업이 사라진다.
- 문서가 산발적이라 재추적이 어렵다.
- 작업이 git/버전으로 묶이지 않아 되돌리기·인수인계가 약하다.
- 한 기기/한 세션을 벗어나면 진행 상태를 넘길 방법이 없다.

## 목표

다섯 가지 능력을 추가해 **지속·재개 가능한 다중 에이전트/다중 기기 실행 OS**로 만든다.

1. 지속 자율 실행 2. 레이트리밋·하위 에이전트 회복력 3. 문서 체계 4. Git 프로토콜 5. 교차 에이전트/기기 핸드오프.

## 대상 사용자 / 이해관계자

- **1차**: 스킬 소유자(CHENGHAO QUAN) — Claude Code에서 긴 자율 작업을 돌린다.
- **2차**: ChatGPT/일반 LLM 사용자 — 모델 중립으로 같은 규율을 텍스트로 받는다.
- **결정권자**: 사람(승인 게이트 보유).

## 기대 산출물

- 갱신된 `SKILL.md`(v1.2.0), 매니페스트·README·CHANGELOG.
- 본 `docs/` 체계, live artifact 대시보드.
- git 커밋·태그(`v1.2.0`)·푸시.

## 성공 기준

- 5개 능력이 SKILL.md에 **내부 모순 없이** 반영(특히 §1.1 의미 먼저 ↔ 자율 실행).
- 자율 실행이 **§1.6 안전 게이트를 보존**함을 독립 테스트로 확인.
- 문서·git이 정의된 규칙대로 일관 적용.
- 모델 중립성 유지(각 기능에 비-Claude-Code 환경 동작 명시).

## 실패 기준 (이러면 축소/중단)

- 자율 실행이 게이트를 약화시켜 위험 작업을 자동화한다.
- 핸드오프 설계가 동시성으로 데이터를 손상시킨다.
- "보장"으로 단정했으나 강제 메커니즘이 없는 과대주장.

## 제약

- SKILL.md는 **프롬프트**다(런타임·락·트랜잭션 없음). 메커니즘이 아니라 규율로 표현.
- 모델 중립(Claude/ChatGPT/일반 챗).
- 프레임워크 호환 — 기존 §0~§17 구조 보존.
