# DECISION LOG — v1.2.0

> 상태: 확정 · 날짜: 2026-06-23 · 소유자: Documentarian · 승인: CHENGHAO QUAN

각 결정은 배경·선택지·선택·이유·버린 안·남은 리스크 순. 대안 상세는 [ALTERNATIVES](ALTERNATIVES.md).

## D1 — 자율 실행을 "승인된 계획 범위 내 자율"로 정의

- **배경**: 요청 ①("묻지 말고 끝까지")이 §1.1("의미 먼저, 실행은 나중")·§15("과한 자동화 경계")와 정면 충돌. 적대 검토가 "정체성 전복"으로 지적.
- **선택지**: (a) 스텝 단위 무조건 자율 (b) 승인된 계획 범위 내 자율 (c) 자율 도입 안 함.
- **선택**: (b). 사람이 방향/계획을 한 번 승인 → 그 범위 내 가역 작업만 확인 없이 연속 실행.
- **이유**: §1.1을 폐기하지 않고 "1회 의미 게이트 후 자율"로 양립. 사용자의 실행 지시가 그 승인 역할.
- **버린 안**: (a) 정체성 위반·위험. (c) 요청 미충족.
- **남은 리스크**: 위임 범위 해석 모호 → 허용목록(D4)으로 축소. 승인자: 사람.

## D2 — 핸드오프 대시보드 동시성: 단일 라이터 / commit-as-CAS

- **배경**: 요청 ⑤를 "락 없는 멀티라이터 git 파일"로 구현하면 lost update·머지충돌(적대 검토 최대 결함).
- **선택지**: (a) 멀티라이터 자유쓰기 (b) 단일 라이터 (c) commit-as-CAS.
- **선택**: 기본 (b), 다중 필요 시 (c). "충돌 없음" 주장 삭제 → "충돌을 git으로 표면화 후 규칙 해소".
- **이유**: 프롬프트엔 락이 없다. 조용한 손상보다 보이는 충돌이 안전.
- **버린 안**: (a) 데이터 손상. 남은 리스크: 동시 push 경합(드묾, 충돌로 가시화). 승인자: 사람.

## D3 — 체크포인트 = 마지막 성공 커밋

- **배경**: 요청 ②의 안전한 재개에는 "어디까지 했는지"가 필요. "idempotent"는 입증 불가(적대 검토).
- **선택**: 체크포인트를 **마지막 성공 커밋**으로 고정. 외부 발송·결제는 자동 재개 제외.
- **이유**: git이 부분 멱등성·가시성 제공. 대시보드 텍스트보다 신뢰 가능.
- **남은 리스크**: 커밋 사이 부분작업 → 재개 전 중복 확인 체크리스트. 승인자: 사람.

## D4 — 게이트는 판단보다 허용목록

- **배경**: "되돌릴 수 있나"를 자율 압박 속 모델이 매번 판단하면 회색지대 실수(force-push·publish·migration).
- **선택**: 자율 진입 시 **허용목록** 명시 + §1.6에 경계 예시 표 추가 + "애매하면 멈춘다".
- **이유**: 사전 정의가 즉석 판단보다 견고. RED/GREEN상 priors가 이미 강하나 회색지대·드리프트 대비.
- **남은 리스크**: 허용목록 작성 부담(작을 때 생략 가능). 승인자: 사람.

## D5 — 문서 체계: `docs/NN-*` 트리 채택 + 혼용 금지

- **배경**: 요청 ③. 기존 §6은 평면 14파일. 두 체계 공존 시 산재(적대 검토).
- **선택**: `docs/00-overview … 05-ops · appendix · assets`. §6.1에 구조+매핑, §15에 혼용 금지.
- **버린 안**: 평면 유지(확장성↓). 남은 리스크: 소규모 과설계 → "작으면 평면 허용" 단서. 승인자: 사람.

## D6 — Git: Conventional Commits + 의미단위 + SemVer 태그 + init 가드

- **배경**: 요청 ④. 자동 `git init`은 기존/상위 repo서 무단 변경 위험. 자율 루프+커밋=커밋 폭주 위험.
- **선택**: init 전 기존/상위 repo 확인, 이름 정규화(kebab/ASCII), 의미단위 커밋, `vX.Y.Z` 주석 태그.
- **남은 리스크**: 태그 명명 과거 비일관(`v1.1.0`·`quetzalcoatl--v1.1.0`) → 앞으로 `vX.Y.Z`로 표준화(과거는 보존). 승인자: 사람.

## D7 — 검증: 단일모델 적대검토 + RED/GREEN (실제 교차검증 아님)

- **배경**: §5는 Claude–GPT 교차검증을 권하나 이 환경엔 GPT 미접속.
- **선택**: §5.6대로 단일모델 적대검토 + 독립 RED/GREEN 행동 테스트. "실제 교차검증 아님" 명시 + 사용자용 GPT 프롬프트 제공.
- **이유**: 정직성(§1.4). 남은 리스크: 단일모델 편향 → 사용자 후속 GPT 검토 권장. 승인자: 사람.
